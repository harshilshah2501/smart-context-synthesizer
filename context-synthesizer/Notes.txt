## Two different “sizes” (don’t mix them)

Your report mixes **three** measures:

| Line in report | What it is | This session |
|----------------|------------|--------------|
| **Real input tokens** (541K) | What Anthropic logged on the **last API call** — uncached + cache_read + cache_write | What Claude Code **actually sent** |
| **Final naive est** (926K) | Offline model: *if every user/assistant/tool byte from all 415 turns were kept in context at once* | **Worst-case counterfactual** — not what Claude sends today |
| **Final synth est** (23K) | Offline model: *Layer 1 + fixed ledger + last 10 turns + current prompt* | **Synthesizer-shaped counterfactual** |

So:

- **541K ≠ 926K** because Claude Code does **not** ship the full raw transcript every turn. It ships a managed, cached prefix. The 926K number is “what if we never compacted and kept everything.”
- **97.5% compression** = `(926K − 23K) / 926K` — naive vs synth **estimates**, not “we saved 97% on the bill.”
