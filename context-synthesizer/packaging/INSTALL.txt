Context Synthesizer — direct install package
=============================================

No git. No GitHub. No rclone. No separate API key at setup (Claude Code Max/Pro login forwards auth).

TEAM LEAD (once)
----------------
1. Build or use the provided context-synthesizer-toolkit-*.tar.gz
2. Extract on SharePoint (or upload the .tar.gz and tell devs to extract after sync)
3. Copy team.conf.example → team.conf
4. Edit team.conf → set SYNC_DIR to your synced OneDrive weekly folder
   (ENABLE_PROXY=1 and ENABLE_WEEKLY_CRON=1 are on by default)
5. Share the SharePoint folder link; tell team to click Sync in OneDrive

UBUNTU LINUX
------------
  sudo apt install -y python3 python3-venv
  Use OneDrive sync (e.g. abraunegg/onedrive) OR download tarball from SharePoint.
  Set SYNC_DIR in team.conf to your Linux sync path (often ~/OneDrive/... not Windows path).
  Same command: bash run-setup.sh firstname.lastname

DEVELOPER (once, ~5 min)
------------------------
1. Get the package:
     Windows — SharePoint link → Sync (OneDrive desktop app)
     Ubuntu  — OneDrive Linux client sync, OR download .tar.gz from SharePoint and extract
2. Open terminal in the package folder, e.g.:
     .../ContextSynthesizer/context-synthesizer-toolkit-...
3. Run (use your Azure email local-part as id):

     bash run-setup.sh firstname.lastname

   Example:
     bash run-setup.sh harshil.shah

4. Done. Live compaction runs in the background; optional Monday reports copy to SharePoint.

SMOKE TEST
----------
  systemctl --user status context-synthesizer-proxy
  open http://127.0.0.1:8080/dashboard   # or :8081 if PROXY_PORT set in .env
  # then use Claude Code — live charts update per API call

Optional weekly upload test:
  bash context-synthesizer/scripts/weekly_sync.sh

(After install, toolkit may also live in ~/.local/share/context-synthesizer)

WHAT YOU GET
------------
PRIMARY — Live compaction (default ON, ENABLE_PROXY=1):
- Claude Code routes through local proxy (systemd user service)
- Dreaming v4 compacts history during long sessions (~90-99% smaller shaped context)
- Use Claude Code while logged in to Max/Pro — CLI forwards auth; no extra API key at setup

OPTIONAL — Weekly SharePoint reports (ENABLE_WEEKLY_CRON=1 in team.conf):
- Automatic summary.md + JSONL every Monday for team rollup
