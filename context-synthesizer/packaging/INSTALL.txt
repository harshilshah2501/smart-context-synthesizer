Context Synthesizer — Motadata team package
==========================================

No git. No API key. Claude Max/Pro login forwards auth to the local proxy.

TEAM LEAD — publish (once per release, from dev machine with OneDrive sync)
--------------------------------------------------------------------------
1. Edit packaging/share.conf only if your OneDrive path changes (default is set).

2. From the repo:
     bash packaging/publish-to-sharepoint.sh

   This builds the toolkit and copies to:
     C:\Users\Harshil Shah\OneDrive - Motadata\Context-Synthesizer\
       context-synthesizer-toolkit-latest/    ← always current
       context-synthesizer-toolkit-YYYY.MM.DD.tar.gz

   OneDrive syncs to SharePoint automatically. No install.sh edits per release.

UBUNTU 22.04 — developer install (~2 min)
-----------------------------------------
Prereqs:
  sudo apt install -y python3 python3-venv curl
  claude auth login
  loginctl enable-linger $USER

Get the toolkit (pick one):

  A) OneDrive / SharePoint sync (recommended — no extract)
     Sync the Context-Synthesizer folder, then:
       cd context-synthesizer-toolkit-latest

  B) Download the dated archive from SharePoint
       tar -xzf context-synthesizer-toolkit-YYYY.MM.DD.tar.gz
       cd context-synthesizer-toolkit-YYYY.MM.DD

Install:
  bash run-setup.sh firstname.lastname

Example:
  bash run-setup.sh harshil.shah

Verify:
  csynth doctor
  csynth dashboard

Toggle proxy later (no reinstall):
  csynth proxy on
  csynth proxy off

Use Claude Code normally. Dashboard updates live per API call.

SMOKE TEST
----------
  csynth status
  csynth doctor
  csynth dashboard

TROUBLESHOOTING
---------------
Proxy / API errors:
  csynth restart
  grep ANTHROPIC_BASE_URL ~/.claude/settings.json   # http://127.0.0.1:8080

Port 8080 busy:
  Setup auto-picks 8081. Re-run configure_claude_proxy.sh if you change port.

Disable synthesizer:
  csynth proxy off

  (Re-enable: csynth proxy on)
